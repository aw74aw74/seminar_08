# Менеджер заметок (Notes Manager API)

Приложение для управления личными заметками, разработанное с использованием Spring Boot и Spring Data JPA.
За основу взят проект из семинара 6, дополненный функционалом логирования с использованием Spring AOP.

## Используемые технологии

- Java 17
- Spring Boot 3.4.3
- Spring Data JPA
- Spring AOP
- База данных H2
- Maven

## Возможности приложения

1. Создание новых заметок
2. Просмотр всех заметок
3. Получение заметки по ID
4. Обновление заметок
5. Удаление заметок
6. Логирование действий пользователя через Spring AOP

## Доступные API endpoints

1. `GET /api/notes` - Получить список всех заметок
2. `GET /api/notes/{id}` - Получить заметку по ID
3. `POST /api/notes` - Создать новую заметку
4. `PUT /api/notes/{id}` - Обновить существующую заметку
5. `DELETE /api/notes/{id}` - Удалить заметку

## Как запустить приложение

1. Предварительные требования:
   - Установленная Java 17 или выше
   - Установленный Maven

2. Клонирование репозитория:
   ```bash
   git clone https://github.com/aw74aw74/seminar_08
   cd seminar_08
   ```

3. Запуск приложения:
   ```bash
   mvn spring-boot:run
   ```

4. Доступные адреса после запуска:
   - API: http://localhost:8080/api/notes
   - Консоль базы данных H2: http://localhost:8080/h2-console

## Примеры использования API

### Создание новой заметки
```bash
curl -X POST -H "Content-Type: application/json" -d "{\"title\":\"Важная заметка\",\"content\":\"Содержание заметки\"}" http://localhost:8080/api/notes
```

### Получение всех заметок
```bash
curl http://localhost:8080/api/notes
```

### Получение заметки по ID
```bash
curl http://localhost:8080/api/notes/1
```

### Обновление заметки
```bash
curl -X PUT -H "Content-Type: application/json" -d "{\"title\":\"Обновленная заметка\",\"content\":\"Новое содержание\"}" http://localhost:8080/api/notes/1
```

### Удаление заметки
```bash
curl -X DELETE http://localhost:8080/api/notes/1
```

## Настройки базы данных

- JDBC URL: `jdbc:h2:mem:notesdb;DB_CLOSE_DELAY=-1;DB_CLOSE_ON_EXIT=FALSE`
  - `DB_CLOSE_DELAY=-1` - предотвращает закрытие базы данных при отключении последнего соединения
  - `DB_CLOSE_ON_EXIT=FALSE` - предотвращает закрытие базы данных при завершении работы JVM
- Пользователь: `sa`
- Пароль: ` ` (пустой)
- Драйвер: `org.h2.Driver`

## Логирование действий

В приложении реализовано автоматическое логирование всех действий с заметками с использованием Spring AOP.
Для каждого вызова метода сервиса в консоль выводится следующая информация:
- Время вызова метода
- Название вызванного метода
- Параметры вызова
- Результат выполнения
- Время выполнения операции

Логирование реализовано с помощью аспектно-ориентированного программирования:
- Аннотация `@TrackUserAction` помечает методы, которые нужно отслеживать
- Класс `UserActionAspect` перехватывает вызовы помеченных методов и логирует информацию

Пример лога в консоли:
```
==================================================
Время вызова: 2025-03-22T03:49:45
Вызов метода: NoteService.createNote
Параметры: [Note(id=null, title=Важная заметка, content=Содержание заметки)]
Результат: Note(id=1, title=Важная заметка, content=Содержание заметки)
Время выполнения: 123мс
==================================================
